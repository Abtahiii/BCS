## GitHub-এ কীভাবে Edit করবে (Browser থেকেই)

`github.com` → তোমার repository `BCS`-এ যাও

## নতুন প্রিলিমিনারি প্রশ্ন যোগ করার পদ্ধতি

`data-international.js` ফাইলে `INTL_PRELIM` array-এর শেষে নিচের format-এ যোগ করো:

```javascript
{
  id: 'ip16',                    // ← unique id (আগেরগুলো ip01-ip15, তাই ip16 দাও)
  topic: 'conflict',             // ← টপিক (নিচের তালিকা দেখো)
  q: 'প্রশ্নের টেক্সট এখানে লিখবে',
  opts: [
    'প্রথম অপশন',               // ← index 0
    'দ্বিতীয় অপশন',             // ← index 1
    'তৃতীয় অপশন',              // ← index 2
    'চতুর্থ অপশন',              // ← index 3
  ],
  ans: 2,                        // ← সঠিক উত্তরের index (0=প্রথম, 1=দ্বিতীয়...)
  exp: 'কেন এটি সঠিক উত্তর তার ব্যাখ্যা'   // ← (ঐচ্ছিক)
},
```

### টপিক তালিকা (আন্তর্জাতিক):
- `conflict` → সংঘাত ও যুদ্ধ
- `geo` → ভূরাজনীতি
- `regional` → আঞ্চলিক সহযোগিতা
- `economy` → বৈশ্বিক অর্থনীতি
- `climate` → জলবায়ু ও পরিবেশ
- `tech` → প্রযুক্তি ও AI
- `org` → আন্তর্জাতিক সংগঠন

### টপিক তালিকা (বাংলাদেশ):
- `constitution` → সংবিধান ও রাজনীতি
- `history` → মুক্তিযুদ্ধ ও ইতিহাস
- `economy` → অর্থনীতি ও উন্নয়ন
- `geography` → ভূগোল ও পরিবেশ
- `society` → সমাজ ও সংস্কৃতি
- `governance` → শাসন ও প্রশাসন

## নতুন লিখিত প্রশ্ন ও উত্তর যোগ করার পদ্ধতি

`data-international.js` ফাইলে `INTL_WRITTEN` array-এর শেষে যোগ করো:

```javascript
{
  id: 'iw08',           // ← unique id (আগেরগুলো iw01-iw07, তাই iw08 দাও)
  topic: 'economy',     // ← উপরের টপিক তালিকা থেকে
  marks: 10,            // ← প্রশ্নের নম্বর
  q: 'প্রশ্নের টেক্সট এখানে লিখবে?',
  answer: `উত্তর এখানে লিখবে।

একাধিক প্যারাগ্রাফ লিখতে পারবে।

backtick ব্যবহার করলে line break ঠিকঠাক কাজ করবে।

ভূমিকা:
এভাবে বিভিন্ন section করতে পারবে।

উপসংহার:
শেষে উপসংহার লিখবে।`
},
```

**⚠️ গুরুত্বপূর্ণ:** `answer` ফিল্ডে সবসময় backtick (`` ` ``) ব্যবহার করো, সাধারণ quote (`'` বা `"`) নয়।

একই নিয়ম, শুধু `data-bangladesh.js` ফাইলে:

## সাধারণ ভুল ও সমাধান

### ভুল ১: Site update হচ্ছে না
✅ সমাধান: Browser-এ Ctrl+Shift+R চাপো (hard refresh)

### ভুল ২: সাইট ভেঙে গেছে / কিছু দেখাচ্ছে না
✅ সমাধান: GitHub-এ "History" থেকে আগের version-এ ফিরে যাও:
`Code` → `Commits` → ভালো version → `...` → "Revert"

### ভুল ৩: Comma ভুলে গেছি
✅ Data ফাইলে প্রতিটি `{ }` block-এর শেষে comma (`,`) দিতে হবে।
শেষের টাতে comma না দিলেও চলে, কিন্তু দিয়ে রাখলে ভালো।

### ভুল ৪: id duplicate হয়ে গেছে
✅ প্রতিটি প্রশ্নের id অবশ্যই আলাদা হতে হবে।
আন্তর্জাতিক প্রিলিমিনারি: ip01, ip02... (যেটা শেষে আছে তার পরের নম্বর দাও)
বাংলাদেশ প্রিলিমিনারি: bp01, bp02...
আন্তর্জাতিক লিখিত: iw01, iw02...
বাংলাদেশ লিখিত: bw01, bw02...
